거부환경 해상 COP 의미 전송 시스템

통신망이 불안정할 때 원본 데이터를 모두 보내려 하면 중요한 판단이 늦어집니다. 이 프로젝트는 AIS, SAR, 기상, OSINT, 네트워크 상태를 작전적으로 의미 있는 이벤트로 줄여서 지휘관 화면에 남기는 방식을 보여줍니다.

용어 도움말 · 모르는 줄임말을 눌러보세요

한 문장으로 설명하면

“무전이 잘 안 되는 상황에서, 모든 영상을 보내는 대신 꼭 필요한 작전 의미만 먼저 보내는 시스템”입니다.

예를 들어 4MB짜리 원본 SAR 이미지와 AIS 로그를 다 보내지 못해도, “어느 선박의 AIS가 끊겼고, 같은 시간대 SAR에는 선박 크기 물체가 보이며, 이 정보는 어떤 근거로 판단했는지”라는 1KB 안팎의 경보 카드는 보낼 수 있습니다.

왜 이게 T3 주제인가

COP / C2 지휘관이 보는 상황도는 단순 지도가 아니라 “무엇이 중요하고 왜 믿을 수 있는지”를 보여줘야 합니다.
Battle Network 통신 링크가 약해지면 원본 데이터보다 우선순위가 높은 의미 메시지를 먼저 보내야 합니다.
Maritime Domain 데이터는 T4 해상 MDA에서 가져오지만, 핵심 산출물은 T3식 C2/semantic transmission입니다.

비유로 이해하기

택배 박스가 아니라 송장부터 보낸다

원본 데이터는 큰 택배 박스와 같습니다. 통신망이 좁으면 박스 전체를 못 보냅니다. 그래서 먼저 송장처럼 “누가, 어디서, 무엇을, 왜 중요한지”를 보냅니다.

응급실 triage와 비슷하다

모든 환자를 동시에 볼 수 없을 때, 생명이 위험한 환자를 먼저 봅니다. 이 시스템도 모든 데이터를 똑같이 보내지 않고, 작전적으로 급한 이벤트를 먼저 보냅니다.

Priority Routing

통신이 제한되면 모든 것을 보낼 수 없습니다. 그래서 이벤트마다 점수를 계산합니다.

priority = mission impact + urgency + confidence + trust + novelty + network efficiency
Network mode전송 판단데모에서 보이는 결과

semantic_summarySEND 핵심만상위 경보 카드만 살아남음

발표할 때 쓸 수 있는 설명

“저희는 해상 데이터를 많이 보여주는 대시보드를 만든 것이 아닙니다. 통신이 나빠졌을 때도 지휘관에게 필요한 작전 의미가 살아남도록, 원본 데이터를 증거가 붙은 작은 이벤트 메시지로 바꾸는 COP 시스템을 만들었습니다.”

“AIS는 진실이 아니라 하나의 주장입니다. 그래서 SAR, 기상, OSINT, 네트워크 상태와 함께 evidence bundle로 묶고, 신뢰도와 freshness를 표시합니다.”

한계와 안전장치

이 데모는 실제 작전 판단 시스템이 아닙니다. 합성 데이터로 시스템 패턴을 보여주는 것입니다. AIS가 끊겼다고 불법이라고 판단하지 않으며, SAR detection도 사람의 검토가 필요한 후보로만 다룹니다.

API key, 계정, 원본 민감 데이터는 리포트와 데모에 포함하지 않습니다.
