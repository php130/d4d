# Hackathon Submission Security Checklist

이 체크리스트는 D4D 해커톤 규칙을 제출물 제작 관점으로 바꾼 것입니다. 발표자료, GitHub/GitLab, 데모 영상, 심사용 제출물 배포 전 확인합니다.

## Must Not Include

- [ ] 기밀·군사보안·민감정보
- [ ] 개인정보(PII), 얼굴 식별 정보, 연락처, 계정 정보, 위치 정보
- [ ] 원본 데이터베이스(DB) 또는 원본 raw data
- [ ] API Key, Access Token, 인증정보, cookie, session token
- [ ] 불법적으로 획득한 정보
- [ ] 악성코드, 침투 도구, 서비스 방해 코드
- [ ] 비인가 시스템 접근을 유도하는 절차

## Data Handling

- [ ] 외부 연결 데이터는 해커톤 목적 범위 내에서만 사용했다.
- [ ] 제출물에는 최소 샘플 데이터만 포함했다.
- [ ] 원본 데이터는 제거하거나 익명화했다.
- [ ] 시연 데이터는 비식별화/마스킹 처리했다.
- [ ] 데이터 출처, 접근 조건, 수집 시점을 기록했다.
- [ ] raw data를 외부 공개 저장소에 업로드하지 않았다.

## Demo Review

- [ ] 화면에 실제 개인 이메일, 전화번호, 계정명, 위치정보가 보이지 않는다.
- [ ] 민감한 조직/개인의 실제 피해 가능성을 높이는 표현을 피했다.
- [ ] 리스크 점수는 단정적 혐의가 아니라 조사 우선순위로 표현했다.
- [ ] 플랫폼 화면 캡처에 token, workspace URL, 비공개 데이터가 노출되지 않는다.

## Responsible Development

- [ ] 실제 환경에 영향을 주는 테스트를 하지 않았다.
- [ ] 비인가 시스템 접근, 우회, 서비스 방해, 악성코드 배포가 없다.
- [ ] 프로젝트 목적이 연구, 분석, 훈련, 방어 범위에 있다.
- [ ] 오픈소스/외부 코드/데이터의 라이선스와 출처를 기록했다.
